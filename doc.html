Documentation - Api

Routes

[*] GET /users

Returns a list of all users from the database or 204 if the user is missing

[*] GET /users/:id

Requires: user id that can be found when calling /users
Returns: information for specific user based on the provided ID if it exits.
Error codes:
  • 500: when something goes wrong while fetching data
  • 204: no data was found

POST /users

Creates a new user in the database.

Requires: username, password, name, email.
Returns: User information that was inserted into database.
  • 400: missing input fields
  • 500: Something going wrong when trying to create user

[*] PUT /users/:id


Requires: username, password, name, email.
Returns: updated user information that was inserted into database.
  • 200: user updated succesfully
  • 400: missing input fields
  • 500: Something going wrong when updating

POST /login

Login in a user.

Requires: username, password.
Returns: jwt token with user info inside.
  • 200: logged in succesfully
  • 401: wrong username or password
  • 500: Something going wrong when trying to login

[*] = Requires logging in, Bearer token must be provided
